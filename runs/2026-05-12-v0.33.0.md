## Piece 1: PowerShell tool

`tools.py`: `tool_run_shell` now branches on `CONFIG.use_powershell`
(set via CLI `--powershell` or env `OPEN_CODE_USE_POWERSHELL=1`).
When enabled on Windows, routes the command through pwsh:

```python
subprocess.run([ps_exe, "-NoProfile", "-NonInteractive", "-Command", command],
               shell=False, ...)
```

`pwsh` (PowerShell 7+) is preferred over legacy `powershell.exe`;
falls back gracefully to cmd if neither is found. `-NoProfile`
skips user profile load (faster + reproducible). `-NonInteractive`
prevents credential prompts from hanging the agent loop.

Verified live (`tests/_live_pwsh_check.py`):

  - PowerShell path: `Write-Output "hello from pwsh"; Get-Date -Format yyyy-MM-dd`
    -> stdout "hello from pwsh\n2026-05-12\n", exit 0.
  - cmd path (env unset): same command -> "'Write-Output' is not
    recognized" -- correct cmd behavior; confirms we're not silently
    falling through to PowerShell when disabled.

CLI flag + Config slot:

```python
parser.add_argument("--powershell", action="store_true", ...)
CONFIG.use_powershell = bool(getattr(args, "powershell", False))
```

No-op on POSIX -- `/bin/sh` already handles bash-flavored commands.

## Piece 2: Integration probe (the bigger piece)

`tests/probe_integration.py`: 5 tests that drive run_loop against a
**FakeLLMClient** which records every kwargs dict it sees and emits
scripted responses. The probe catches the cross-cutting bugs that
adapter unit tests miss -- exactly the v0.30.2 regression class.

| # | Assertion | The bug this catches |
|---|---|---|
| 1 | After iter 1, history[2].role == "tool" | v0.30.3: OpenAI Responses adapter drops user-role tool_results |
| 1 | iter2.history[2].parts[0].tool_call_id == EXPECTED | v0.30.2: tool_use_id not round-tripping (Anthropic 400) |
| 1 | every tool.parameters.type is lowercase | v0.30.2: UPPERCASE Gemini schema rejected by Anthropic/OpenAI |
| 2 | Same invariants via streaming path | Catches stream-vs-non-stream divergence in run_loop |
| 3 | Every built-in TOOL_DECLARATIONS uses lowercase JSON Schema | Catches new tool decls slipping in with UPPERCASE |
| 4 | settings.llm.model wins over DEFAULT_MODEL | v0.30.2: model resolution chain |
| 5 | tool_result Message round-trips via sessions.content_to_dict | Catches JSONL persistence of tool_call_id + role="tool" |

One bug surfaced + fixed in writing the probe:

The `FakeLLMClient.ask` recorder did `dict(kwargs)` -- shallow copy.
The `messages` value is a list reference that run_loop mutates
between iterations. So `fake_llm.calls[0]["kwargs"]["messages"]`
inspected after run_loop finished showed the FINAL state, not the
state at iter-1 time. Snapshot fix:

```python
snapshot = dict(kwargs)
if "messages" in snapshot:
    snapshot["messages"] = list(snapshot["messages"])  # at-call snapshot
if "tools" in snapshot:
    snapshot["tools"] = list(snapshot["tools"])
self.calls.append({"method": "ask", "kwargs": snapshot})
```

This is a pattern bug worth remembering: when intercepting
mid-loop state via instrumentation, the captured dict's mutable
values are still references to the loop's working state.

## Honest trade-offs

- **--powershell is Windows-specific.** On POSIX, the flag is
  accepted but has no effect (pwsh would also work if installed,
  but `/bin/sh` already does what users expect). The README
  documents this.

- **Integration probe uses FakeLLMClient, not real adapters.** The
  point is to catch cross-cutting agent-loop bugs (role, id
  round-trip, tool-decl shape) -- the LAYER where v0.30.2's six
  bugs lived. Bugs in the adapter <-> SDK <-> wire layer still need
  the `_live_*_check.py` smokes; this probe doesn't replace them,
  it catches the structurally-different bugs that are CHEAP to
  trigger but EXPENSIVE to debug live.

- **The probe asserts on internal API shape, not user-visible
  behavior.** If the agent loop gets refactored, this probe will
  break. That's the point -- the contracts asserted (history shape,
  role discipline, lowercase JSON Schema, tool_call_id round-trip)
  are LOAD-BEARING semantics. A break here is a signal to update
  the probe deliberately, not work around it.

- **Test 4 inlines the cli's model-resolution logic** rather than
  calling cli.main. Direct call to cli.main needs a full argv +
  settings setup that's noisy. The inline test verifies the chain
  semantics; an end-to-end CLI test would be redundant with the
  existing live probes' real REPL runs.

## What this unlocks

For the next refactor:
  - Trying to change role discipline -> probe breaks immediately
  - Adding a tool decl with UPPERCASE schema -> probe breaks
  - Skipping tool_call_id round-trip -> probe breaks
  - Forgetting to honor settings.llm.model -> probe breaks

The v0.30.2 wave (Anthropic REPL finding 6 bugs) takes about an
hour of live-testing + key-cost + careful triage. This probe runs
in 0.2s and finds all six instantly.
