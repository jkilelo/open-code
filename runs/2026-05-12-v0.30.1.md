## Adapter bugs caught + fixed

**1. Anthropic adaptive thinking scope was wrong.**

My background research report (`runs/2026-05-12-llm-design.md` and
the original gathered docs) claimed Haiku 4.5 supports
`thinking={"type":"adaptive"}`. The wire disagreed:

```
anthropic.BadRequestError: 400 - {'type': 'error', 'error':
{'type': 'invalid_request_error',
 'message': 'adaptive thinking is not supported on this model'}}
```

Fix: narrowed `_ADAPTIVE_THINKING_MODELS` in `llm/_anthropic.py`
from 4 model substrings to 3:

```python
# WAS
_ADAPTIVE_THINKING_MODELS = (
    "opus-4-7", "opus-4-6", "sonnet-4-6", "haiku-4-5",
)
# NOW
_ADAPTIVE_THINKING_MODELS = (
    "opus-4-7", "opus-4-6", "sonnet-4-6",
)
```

Haiku 4.5 falls through to `{"type":"enabled","budget_tokens":N}` mode
and works fine on the next try. Honest: I should have caught this
before shipping v0.30.0; "the research said so" isn't verification.
The lesson is "live-test before claiming support."

**2. `DEFAULT_MODELS["openai"]` referenced a model that isn't GA.**

My research report listed `gpt-5.5-mini` as the latest cheap default.
Live `client.models.list()` on the user's account returned `gpt-5-mini`,
`gpt-5.4-mini`, `gpt-5-nano`, and `gpt-5-nano`, but no `gpt-5.5-mini`.
Without verification it would have failed for every fresh user.

Fix in `llm/factory.py`:

```python
# WAS
"openai":      "gpt-5.5-mini",
"openai_chat": "gpt-5.5-mini",
# NOW
"openai":      "gpt-5-mini",     # gpt-5.5-mini not yet GA on standard tier
"openai_chat": "gpt-5-mini",
```

## Honest trade-offs

- **gpt-5 reasoning-token budget gotcha** isn't a bug, but worth
  documenting somewhere user-facing. With `thinking_effort=None`
  (default), gpt-5-mini still allocates ~128 reasoning tokens. If
  the caller sets `max_output_tokens=256`, only ~128 are left for
  text -- structured-output responses can come back empty. The
  `_live_openai_check.py` test 4 surfaces this and uses
  `thinking_effort="minimal"` + `max_output_tokens=1024` to dodge
  it. v0.31 could add a runtime warning in `_openai.py` when
  `max_output_tokens` is set < ~512 on reasoning models.

- **gpt-5.5 family** (mentioned in OpenAI's reasoning guide) is not
  yet broadly available. Users on Tier 4+/Scale may have it. The
  factory accepts any model name via `settings.llm.model`; the
  default is just the safest cheap fallback.

- **No Bedrock / Vertex / Azure variants tested.** `AnthropicBedrock`,
  `AnthropicVertex`, `AzureOpenAI` all share the same SDK surface but
  routing is different. v0.31 candidate.
