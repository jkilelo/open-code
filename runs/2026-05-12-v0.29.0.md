## What changed

The agent loop no longer imports `google.genai`. Instead it depends
on a neutral `LLMClient` Protocol declared in a new `llm.py`. The
Gemini-specific translation lives in `llm_gemini.py` (lazy-loaded by
the factory). Swapping providers is now a single
`settings.llm.provider` change.

### New files
- `llm.py` (380 LOC) -- neutral types (`Part`, `Message`, `ToolDecl`,
  `Usage`, `AskResult`, `StreamChunk`), the `LLMClient` protocol,
  exceptions (`LLMError`, `LLMConfigError`, `LLMCallError`), and the
  `make_llm_client(...)` factory with lazy-import dispatch.
- `llm_gemini.py` (340 LOC) -- the only file in the repo that imports
  `google.genai`. `GeminiClient` translates neutral types <->
  `google.genai.types` on every call.
- `tests/probe_llm.py` -- protocol contract probe. Verifies neutral
  types, storage round-trip, factory error modes, structural duck-
  typing, and that `llm.py` has no top-level google import (the
  lazy-provider promise).

### Modified
- `open_code.py` -- `run_loop` now takes `llm: LLMClient | None` and
  builds it lazily via `_make_llm_from_settings`. `_stream_iter_response`
  iterates `LLMClient.ask_stream(...)` (no more `client.models.
  generate_content_stream`). Tool-call dispatch reads `Part.tool_name` /
  `Part.tool_args` and writes `Part.make_tool_result(...)`.
  `_handle_request_specialist` + `_handle_find_specialist` +
  `_handle_delegate_call` accept `llm_client` and pass it through to
  agent_embed / nested run_loop calls.
- `sessions.py` -- `content_to_dict` / `dict_to_content` now operate on
  `Message` / `Part`. JSONL on-disk shape stayed identical for back-compat;
  added `extra` round-tripping (with base64 encoding for bytes values).
- `cli.py` -- builds a single `LLMClient` up-front via
  `_make_llm_from_settings`, surfacing `LLMConfigError` cleanly. Passes
  it to `run_loop` and `run_repl`.
- `repl.py` -- accepts `llm: LLMClient | None`, threads it through all 5
  `run_loop(...)` call sites. `/compact` was using a raw
  `genai.Client(...)`; switched to the neutral `llm.ask(...)`.
- `agent_embed.py` -- added `make_llm_embedder(llm_client)` for
  semantic search via the protocol. `make_genai_embedder` retained
  for the legacy direct-api-key path.

### Probes updated (10)
- `probe_resume_bloat.py`, `probe_concurrency.py`, `probe_stream_error.py`,
  `probe_interleaved.py`, `probe_streaming.py`, `probe_modes.py`,
  `probe_print_mode.py`, `probe_atomic_turn.py`, `probe_sticky_permissions.py`,
  `probe_session_perf.py`, `probe_tier2_batch_a_rest.py` -- all migrated
  off `google.genai.types` and onto neutral `Message`/`Part`. Probes that
  stubbed `genai.Client.models.generate_content_*` now stub `LLMClient.ask`
  / `LLMClient.ask_stream`. Net effect: the hermetic tests now exercise
  the new protocol surface directly.

## Architecture (the promise we made the user)

```
open-code (run_loop, repl, cli, probes)
        |
        | only talks via:
        v
+------------------+
|   llm.py         |   neutral types + Protocol + factory
+------------------+
        |
        | factory.make_llm_client(provider="..."):
        |   lazy-import the right adapter
        v
+---------+--------+-------+--------+
|         |        |       |        |
v         v        v       v        v
gemini  anthropic openai ollama   (your provider here)
```

The user can now switch providers by editing
`.open-code/settings.json`:

```json
{
  "llm": {
    "provider": "anthropic",
    "api_key_env": "ANTHROPIC_API_KEY"
  }
}
```

`open-code` doesn't know or care which adapter answered. The agent
loop, tool dispatch, session storage, autobuild, REPL slash commands,
and all 49 probes work identically across providers (assuming the
adapter implements the three protocol methods).

## Gemini-specific thought_signature handling

A first attempt at a live tool-call run failed mid-loop:

```
LLMCallError: gemini generate_content failed: ClientError: 400
INVALID_ARGUMENT. Function call is missing a thought_signature
in functionCall parts.
```

Gemini's thinking-on-tool-call models REQUIRE the model's
`thought_signature` bytes blob on the followup `function_call` part
that the agent loop sends back. The neutral type wasn't carrying it.

**Fix:** added `Part.extra: dict[str, Any]` -- an opaque
provider-stash. Gemini's adapter writes `extra["thought_signature"]`
on the way in (in `_from_content` and `_to_stream_chunk`) and
restores it on the way out (in `_to_content`). The agent loop never
inspects `extra` -- it just round-trips it. Storage (`sessions.py`)
base64-encodes byte values so resume preserves the state across
restarts.

This is the v1 of the escape-hatch pattern. Other providers can use
`Part.extra` for their own opaque state (Anthropic's `cache_control`,
OpenAI's `tool_call_id`, etc) without polluting the core types.

## Honest trade-offs

- **Pyright noise.** The user's Python environment doesn't have a
  proper site-packages config for the open-code dir, so Pyright sees
  every cross-module import as `Unknown`. These warnings are
  cosmetic; runtime is clean and all 49 probes pass. Could be fixed
  with a `pyrightconfig.json` but that's out of scope for this PR.
- **`Part.extra` is an escape hatch by design.** It carries opaque
  provider-specific bytes/strings. A future provider that needs
  cross-provider semantics (e.g. all of them need turn-IDs) should
  promote that into a first-class field; we'll cross that bridge
  when 3+ providers want it.
- **No anthropic/openai/ollama adapter yet.** The factory will raise
  `LLMConfigError` cleanly if you try those, with a "install the X
  package" message. v0.30+ will land at least Anthropic alongside
  the docs.
- **Old JSONL session files load unchanged.** The on-disk shape was
  always neutral (`{type: "text"/"function_call"/"function_response"}`).
  This refactor swapped the in-memory representation; storage was
  already provider-agnostic.

## Decision

[OK] **Ship v0.29.0.** Decoupled LLM provider from the agent loop
via the neutral `LLMClient` protocol. Factory + lazy-imports +
provider-specific opaque metadata. 49/49 probes; ASCII pure; live
multi-turn + resume tests pass on Gemini.
