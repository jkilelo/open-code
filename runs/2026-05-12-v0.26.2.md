## Fixes shipped

### B2 -- structural path safety

`agent_builder.build_agent`: replaced
```python
if not str(resolved).startswith(str(root_resolved)):
```
with
```python
if not resolved.is_relative_to(root_resolved):
```

`is_relative_to` compares Path components, not string prefixes. The
name regex `^[a-z][a-z0-9-]*$` already blocks traversal, so this
triggers only on regex regression -- which is exactly when a layered
defense should hold.

Test (probe_agent_extensions Test 13): asserts the source of
`build_agent` contains `is_relative_to` AND does NOT contain
`startswith(str(root_resolved))`. Future regression would fire.

### Y3 -- pending duplicate overwrite

Two-pronged fix in `build_agent`:

1. **existing_names augmentation:** before validation, glob
   `.pending/*.md` and add those stems to the `existing_names` set.
   The validator's `_unique_name` then sees pending names and
   suffixes collisions with `-2`, `-3`, ...
2. **archive-before-overwrite:** removed the `auto_approve and ...`
   guard around `_archive_existing(...)`. Now any pre-existing file
   at the target path -- live OR pending -- gets archived before
   the overwrite. Belt + braces.

Tests (probe_agent_extensions Tests 14, 15):
- Two consecutive `request_specialist` calls with the same stub
  response produce two pending files (`-1` and `-2`), the first is
  preserved unchanged.
- Manual `_archive_existing` on a planted pending file produces a
  matching `.history/<name>/<ts>.md` entry.

### Y4 -- sidecar orphan cleanup

`agent_embed.ensure_embeddings`: restructured.

Before:
```python
if stale:
    try:
        ...
        for stale_key in [k for k in sidecar if k not in current]:
            sidecar.pop(stale_key, None)
        _save_sidecar(cwd, sidecar)
    except Exception:
        pass
```

After:
```python
# Y4 fix: prune orphans BEFORE checking which agents are stale
orphans_removed = False
for stale_key in [k for k in sidecar if k not in current_names]:
    sidecar.pop(stale_key, None)
    orphans_removed = True

# ... compute stale + recompute fresh ...
stale_updates_made = ...

# Persist if EITHER condition
if orphans_removed or stale_updates_made:
    _save_sidecar(cwd, sidecar)
```

Test (probe_agent_extensions Test 16): plant 2 agents, embed both,
delete one, call `ensure_embeddings` with an `_emb_never_called`
embedder (raises on any call). Assert the deleted agent's sidecar
entry is gone AND the embedder was never called. Without Y4 fix,
the orphan would survive.

## Honest trade-offs

- **Three brutal-review cycles + three "fix cycle" releases:** v0.26.0
  -> v0.26.1 closed B1+Y1+Y2. v0.26.1 -> v0.26.2 closes B2+Y3+Y4.
  Pattern holds: each cycle catches real bugs, each fix introduces
  one or two more. Diminishing but non-zero. Worth another review
  pass; could stop here too.
- **probe_agent_extensions Test 13 inspects source text** for the
  literal "is_relative_to" / "startswith" strings. This is a brittle
  test in principle (a refactor renaming the variable could break
  it). It's the cleanest assertion of the structural property; the
  alternative is to construct an actual traversal-shaped name and
  verify rejection, but the regex blocks that path before it reaches
  the safety check.
- **Y3 archive-on-any-overwrite** runs `_archive_existing` for every
  pre-existing file at the target path. Most calls hit nothing
  (dedup already ensured a fresh name). Cost is one stat() call per
  build. Negligible.
- **Y4 cleanup runs on every `ensure_embeddings` call**, even when
  no agents have changed. Cost: one set comprehension + one
  conditional sidecar write. Negligible at typical library size.
