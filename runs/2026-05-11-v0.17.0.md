## Feature shipped

### #12 Atomic-commit per turn -- `/undo`

Companion to v0.16's shadow-git checkpointing (#11). v0.16 gave us
turn-START snapshots before each turn; v0.17 adds turn-END snapshots
after each turn AND a `/undo [N]` REPL command that rewinds to the
start of the Nth-most-recent turn.

**Wiring:**
- `run_loop`'s `finally:` block (open_code.py) takes a `turn-end`
  snapshot when `settings.auto_checkpoint` is True. Lives in
  `finally` so it fires even on `KeyboardInterrupt` and on the
  exception path -- the user-visible promise is "after the loop
  returns, we have a snapshot of whatever you ended up with."
- `SessionStore.recent_checkpoints(session, phase=None, limit=20)`
  reads the JSONL and returns recent `checkpoint` events
  newest-first, optionally filtered by phase.
- REPL `/undo [N]`:
  1. Looks up the Nth-most-recent `turn-start` checkpoint event
  2. Shows `git diff --stat <target>..HEAD` so the user sees what
     they're about to lose
  3. Asks them to literally type `undo` to confirm
  4. Takes a `pre-undo-from <sha>` safety snapshot (so they can
     roll forward if they change their mind)
  5. Calls `checkpoints.restore(cwd, target)`

**Why turn-END too:**
- Together with turn-start they bracket the work each prompt did,
  which makes "diff the most recent turn" trivial.
- Survives mid-stream errors (the `finally` block runs even when
  the loop raises) so the user can see "what state ended up on
  disk when my Ctrl-C interrupted you."

## Honest trade-offs

- **`/undo` is per-session, not per-CWD-history.** It looks up
  turn-start checkpoints in the current session's JSONL only. If
  the user switches sessions (`/clear` or `/switch`), prior session
  turn-starts are invisible to `/undo`. (They are still listed by
  `/checkpoints`, which reads the shadow repo's `git log`.)
- **`/undo` requires `auto_checkpoint=True`.** Without it, no
  turn-start events exist to rewind to. Acceptable: this is a
  power-user feature gated by an explicit flag, exactly like
  Claude Code's behavior.
- **Two snapshots per turn doubles shadow-repo size growth.** Same
  mitigation as v0.16: future `/checkpoints prune` or `git gc`
  automation. For now, `rm -rf .open-code/checkpoints.git` resets.
- **No automatic rollback on tool error.** I considered: "if the
  model crashes mid-edit, auto-restore turn-start." Rejected because
  most tool errors are recoverable and the model retries; restoring
  silently would lose successful edits. The user opts in to undo
  by typing `/undo`.
- **`turn-end` runs even after auth/import errors that happen before
  the loop body.** The `finally` will try to snapshot even if we
  never ran the model. That's intentional: a baseline snapshot of
  the state at exit is still useful for forensic comparison against
  `turn-start`.
