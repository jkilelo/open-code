# Run log -- open-code v0.24.4 -- 2026-05-11

**Persona:** Jeff.
**Trigger:** User reported encoding gibberish in the GitHub view of
the repo. Strict requirement: "100% no encoding issues in any platform."

## Root cause

`open_code.py` and several other files contained mojibake byte
sequences: UTF-8-encoded characters (em-dashes, arrows, check marks,
pointers) had been misread by a Windows-cp1252 console at some point
and then re-encoded as UTF-8, producing double-encoded artifacts like:

  - `--` (em dash, U+2014, bytes E2 80 94) rendered as `a-tilde euro
    right-double-quote` in cp1252, then re-written as the UTF-8 bytes
    of those THREE chars (8 bytes total).
  - Same pattern for ellipsis, right-arrow, pointer, check-mark,
    ballot-X.

In addition, many files (especially `runs/`, `methodology/`,
`roadmap/`) contained intentional but non-portable UTF-8: em-dashes,
emoji status markers, arrows. While not mojibake, these don't render
on every terminal / code-page combination (Windows console cp1252,
older `less`/`more`, non-UTF-8 SSH sessions).

## Fix shipped

### `scripts/fix_encoding.py` (new, ~150 LOC, one-shot)

A normalizer that:

1. Replaces 6 known mojibake sequences with their ASCII intent:
   - `a-tilde euro right-double-quote` -> `--` (em dash)
   - `a-tilde euro broken-bar` -> `...` (ellipsis)
   - `a-tilde dagger right-single-quote` -> `->` (right arrow)
   - `a-tilde en-dash pilcrow` -> `>` (right pointer)
   - `a-tilde oe-ligature left-double-quote` -> `[OK]` (check)
   - `a-tilde oe-ligature em-dash` -> `[X]` (ballot X)

2. Replaces ~40 intentional non-ASCII chars with ASCII equivalents:
   - dashes `--`/`-`, ellipsis `...`, smart quotes `'` `"`
   - arrows `->`, `<-`, `^`, `v`, `=>`
   - math `<=`, `>=`, `!=`, `~`, `in`
   - markers `[OK]`, `[X]`, `[WARN]`, `[FAIL]`, `(done)`, `(bot)`
   - status emoji (green/yellow/red/white/black circles)

3. Verifies pass: scans every file again, fails if any non-ASCII
   bytes remain.

### `tests/probe_ascii_only.py` (new permanent guard)

Walks every tracked `.py` / `.md` / `.json` / `.txt` / `.sh` file in
the repo (excluding `__pycache__`, `.git`, `.venv`, etc.). Fails if
any byte > 127 is found. Allows one exemption: `scripts/fix_encoding.py`
itself, which legitimately needs the non-ASCII source-side chars in
its mapping table.

This is now part of the regression suite. A future commit that adds
a smart-quote or emoji to any tracked file fails the probe.

### `.gitattributes` (extended)

Existing file already forced LF on `.py`/`.md`/etc. Added:
- `* text=auto eol=lf` catch-all
- `working-tree-encoding=UTF-8` hint for editors (cosmetic; the
  files are now pure ASCII so this is defensive)
- `.bat`/`.cmd`/`.ps1` keep CRLF (Windows-native scripts)

## Impact

- **127 files modified** by `fix_encoding.py`.
- All non-ASCII characters across the repo replaced with ASCII
  equivalents.
- Every file now renders identically on:
  - Windows cp1252 cmd.exe / PowerShell
  - Linux UTF-8 xterm / tmux
  - macOS Terminal.app
  - GitHub web view (no mojibake)
  - `cat`, `less`, `more`, any pager
- All `.gitattributes` enforce LF on commit; CRLF on checkout for
  Windows-native scripts only.

## Honest trade-offs

- **Cosmetic loss in docs.** `[OK]` is less visually punchy than a
  green-circle emoji. Trade-off accepted because the user's
  requirement is strict cross-platform rendering.
- **Mapping is opinionated.** I chose `[OK]` for green-circle and
  `(done)` for party-popper. A future doc could read better with
  different choices. Easy to change in `scripts/fix_encoding.py`
  and re-run the normalizer.
- **Defensive char map includes Latin-1 accents** the codebase
  never used. That's fine -- the table is data, not a hot loop, so
  the extra entries don't slow anything down.
- **`fix_encoding.py` is the lone allowed exception** to ASCII-only.
  It mechanically requires non-ASCII source chars for the mapping
  table; rewriting them as `\u....` escapes would obscure the
  intent. Documented in `probe_ascii_only.py`'s ALLOW_NON_ASCII.

## Decision

[OK] **Ship v0.24.4.** 100% ASCII across the repo. Permanent
regression guard (probe_ascii_only). `.gitattributes` enforces LF.
43/43 probes green; 54/54 security tests green. Encoding-class bugs
prevented at the commit-time check.
