## #3 Skills system

New module `skills.py` (188 LOC). Skills live at:
```
<project>/.open-code/skills/<name>/SKILL.md
```

SKILL.md format:
```
---
name: review-pr
description: Brutal-review a pull request against project standards
allowed-tools: read_file, list_dir, run_shell
disable-model-invocation: false
---
You are reviewing PR $ARGUMENTS.

Project state:
!`git status --short`
!`git diff main --stat | head -40`

Walk the diff...
```

**What happens when the user types `/skill review-pr 1234`:**
1. open-code finds the matching SKILL.md
2. Replaces `$ARGUMENTS` with `1234`, `$1`..`$9` with shlex-split positionals
3. Resolves `` !`cmd` `` blocks by running them in the CWD (20s timeout)
4. The expanded body becomes the user prompt for the next agent turn

Minimal hand-rolled frontmatter parser (no PyYAML dep). Supports
`key: value`, simple lists (comma-separated), booleans. Anything else
becomes a string.

Two new slash commands in REPL:
- `/skills` — list available skills
- `/skill <name> [args]` — invoke a skill with positional args

`disable-model-invocation` flag is parsed and stored on the Skill
object; v0.7 only honors explicit `/skill` invocation. Reserved for
v0.8+ auto-discovery via skill description.

## Honest trade-offs

- **Hand-rolled frontmatter parser**, not full YAML. Skills can't use
  nested objects in frontmatter, just key: value + lists + booleans.
  Adequate for the use case; adds zero deps.
- **No auto-invocation in v0.7.** The `disable-model-invocation`
  flag is stored but unused. Auto-discovery (model decides to invoke
  based on description match) is v0.8+. v0.7 ships explicit
  `/skill` only.
- **`` !`cmd` `` resolves via shell=True** — same posture as
  `run_shell` tool. Subject to the same destructive-command
  denylist? NO — skills run `cmd` outside the tool path, so the
  denylist DOESN'T fire. The trust model: skills are project files
  Jeff wrote himself; if he embedded `rm -rf /` he meant it. Worth
  documenting; defer hardening to v0.8.
- **Skill bodies cap at 50KB.** Above that we truncate with a
  `[...truncated]` marker. Generous enough; smaller than @-file ref
  cap (200KB) because skills are templates, not data.
