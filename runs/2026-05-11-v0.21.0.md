## Feature shipped

### #23 Output styles

Named system-instruction overlays. A style is a short paragraph
appended under `## Output style: <name>` that shifts the model's
TONE without changing its capabilities. Models honor it the same way
they honor any `system_instruction` directive.

**Built-in styles** (in `output_styles.BUILTIN_STYLES`):
- `default` -- no overlay (intentional empty string)
- `concise` -- lead with the answer, skip filler, bullet-points for lists
- `explanatory` -- pre-tool one-liner reasoning + post-tool summary
- `learning` -- name patterns, prefer small steps, treat user as learner
- `pair-programmer` -- propose change before files-larger-than-one
- `yolo` -- make decisions, don't ask, action-first

**Custom styles:**
- `<cwd>/.open-code/output-styles/<name>.md` -- project-local
- `~/.open-code/output-styles/<name>.md` -- user-level
- Project overrides user overrides built-in (same name wins from
  most-specific layer).

**New module: `output_styles.py` (~120 LOC)** -- API:
- `BUILTIN_STYLES: dict[str, str]`
- `list_available(cwd) -> [(name, source)]`
- `resolve_overlay(name, cwd) -> (overlay_text, source_label)`
- `apply_to_system_instruction(base, name, cwd) -> (new_system, source)`

**Wiring:**
- `settings.Settings.output_style: str = "default"` (new field).
- `settings.load_layered_settings` reads `output_style` from JSON.
- `cli.py --style <name>` flag + `--list-styles` flag.
- `cli.main` calls `apply_to_system_instruction` AFTER
  `build_system_instruction_layered` so style overlays sit
  alongside the four-tier memory layers in the prompt.
- REPL `/style [name]` command -- shows current + lists available with
  no arg, sets the value with one arg. (Note: current REPL session's
  system_instruction is fixed at REPL start; `/style` takes effect
  on the next session via `/clear`.)

## Honest trade-offs

- **`/style` in the REPL doesn't apply to the current session.** The
  system_instruction is finalized in cli.main before run_repl runs,
  and it's passed in as a constant. To apply a new style mid-REPL,
  the user `/clear`s (which creates a new Session) -- but that also
  drops conversational context. Future fix: have run_repl rebuild
  the system_instruction per turn from `(base, settings.output_style)`.
- **Built-in overlays are English-only.** No i18n. A multilingual user
  can drop their own custom style file at any of the resolution
  paths; the file body is appended verbatim.
- **No validation that the model actually obeys the overlay.** A
  small model on `yolo` style might still hedge; a large one on
  `concise` might still ramble. The overlay is a *hint*, not a hard
  constraint.
- **Style overlay sits AFTER memory layers in the prompt.** Order:
  base -> project memory -> managed settings (if any) -> style overlay.
  Putting style last means it's the "most recent" instruction, which
  models tend to weight more.
- **Custom style files use markdown extension by convention** but no
  parsing happens -- body is appended raw. No frontmatter, no
  `$ARGUMENTS` interpolation. Style files are plain prose.
