## #9 Architect/editor split (Aider pattern)

Two new `Settings` fields:
- `architect_model: str | None` â€” model used by `/plan` when set
- `editor_model: str | None` â€” model used by `/act` when set

Surfaces:
- `settings.json` â†’ `models: { architect: "...", editor: "..." }`
- CLI flags: `--architect <model>`, `--editor <model>`
- CLI flags override settings.json (consistent with other flags)
- Falls through to `current_model` when either is None (current behavior)

The split composes with the v0.2.1 model fallback chain:
1. `/plan` tries `architect_model`
2. If unavailable, the fallback chain kicks in
3. Loop succeeds (or fails per existing semantics)
