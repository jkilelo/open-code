## Features shipped

### #13 `/compact` slash command

New `/compact [N]` REPL command. Summarizes older session history
into a single synthetic `user` message; keeps the last N (default 10)
verbatim. The original JSONL events are NOT deleted -- a new `compact`
event records the summary and reconstruction params.

JSONL event:
```json
{"kind": "compact",
 "summary": "Files: a.py, b.py. Decided: use pathlib...",
 "kept_recent": 10,
 "dropped": 47,
 "model": "gemini-3.1-flash-lite-preview",
 "ts": "..."}
```

`SessionStore.load_history` now scans for the most recent `compact`
event. If found, it replaces the first `len(all_msgs) -
kept_recent` messages with a single synthetic user message tagged
`[compacted earlier history summary]`. The recent N preserve
verbatim.

REPL invocation:
1. Loads full history (uncapped)
2. If <= N msgs, prints "nothing to compact" and returns
3. Otherwise renders the dropped msgs as plain-text stubs (role: text
   for plain msgs; `[tool name(args)]` and `[tool result name: ...]`
   for tool calls/results)
4. Calls Gemini with "Summarize this prior conversation in 5-10
   sentences" + a tight system instruction
5. Appends the `compact` event with the summary
6. Reloads history in-process so the current REPL session uses the
   compacted form going forward

### #19 Extended @-providers (Continue.dev pattern)

Four new `@<name>` providers in `expand_file_refs`:
- `@diff` -> `git diff` output (capped 30KB)
- `@diff:staged` -> `git diff --staged`
- `@tree` -> `tree -L 2` if installed, otherwise manual depth-2 walk
- `@problems` -> tries `ruff check` / `mypy` / `pyright`; first that
  succeeds wins (capped 10KB)
- `@cwd` -> the absolute CWD path

Providers compose with existing `@<path>` file refs. The output is
prepended to the prompt as `<context kind="...">...</context>`
blocks (not `<file>` -- these aren't file contents).

Unknown `@<name>` tokens pass through as literal text (no false
positives).

## Honest trade-offs

- **Compact summary quality depends on the model.** v0.15.1 calls the
  current_model with a tight system instruction; quality varies by
  model. The probe uses a mock (no live LLM) to test the event-handling
  semantics, not the summary content quality.
- **Compact doesn't replace the original events.** They stay in the
  JSONL for auditability. Only `load_history` skips them. If a user
  ever needs to "uncompact", they can edit the JSONL by hand.
- **`@problems` is best-effort** across linters. If none are installed
  the provider returns "(no linter found...)" -- not an error.
- **`@tree` falls back to a manual walk** if the `tree` binary isn't
  installed. Output looks slightly different but covers the same depth-2.
- **`@diff` is silent if not in a git repo** -- returns None, the
  provider is skipped, no entry in refs. User just sees their literal
  `@diff` text in the prompt.
