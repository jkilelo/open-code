## Feature shipped

### #20 `--print` / `-p` JSON output mode

Non-interactive structured-output mode. Emits one JSON object per
line to stdout for each event in the agent loop. Designed for piping
into other tools, scripting, and IDE integrations.

**CLI:**
```
$ open-code --print "summarize @README.md"
{"type": "session_start", "session_id": "...", "model": "gemini-3.1...", "task": "...", "cwd": "..."}
{"type": "text", "iteration": 1, "text": "I'll read README.md.", "input_tokens": 42, "output_tokens": 18}
{"type": "tool_use", "iteration": 1, "name": "read_file", "args": {"path": "README.md"}}
{"type": "tool_result", "iteration": 1, "name": "read_file", "ok": true, "result": {...}}
{"type": "text", "iteration": 2, "text": "# Summary\n...", "input_tokens": 1024, "output_tokens": 312}
{"type": "session_end", "session_id": "...", "exit_code": 0, "iterations": 2, "input_tokens": 1066, "output_tokens": 330, "tool_calls": 1, "tool_errors": 0, "wall_seconds": 2.881}
```

**Event types:**
- `session_start` -- at run_loop entry: session_id, model, task (truncated 500 chars), cwd
- `text` -- per turn that produced model text: iteration, text, input_tokens, output_tokens
- `tool_use` -- per function call: iteration, name, args (before permission gate runs, so denied calls still emit)
- `tool_result` -- per resolved call: iteration, name, ok, result
- `session_end` -- at run_loop exit: exit_code, iterations, total tokens, tool_calls, tool_errors, wall_seconds

**Wiring:**
- `tools.Config.print_json: bool = False` -- new field. Was missing
  `statusline_on` too (it lived as an attribute set in cli but wasn't
  declared); fixed both in this change.
- `cli.py --print` / `-p`:
  - Forces `args.quiet = True` and `args.no_stream = True` (we emit
    one text event per turn, not chunks)
  - Sets `CONFIG.print_json = True`
- `open_code._emit_json(event_type, **fields)` -- no-op when
  `CONFIG.print_json=False`; otherwise writes a JSON line to stdout
  and flushes. Tolerates broken-pipe / OSError.
- `run_loop` emission points:
  - `session_start` at the top (after client init, before SessionStart hook)
  - `text` after `history.append(model_content)` when the model produced any text
  - `tool_use` inside the function_calls loop before permission eval
  - `tool_result` after tool execution, before PostToolUse hook
  - `session_end` in the `finally:` block after `store.append_end`
- One additional change: the non-streaming `print("".join(emitted))`
  call (which dumps model text to stdout for the human eye) is now
  gated on `not CONFIG.print_json` so we don't mix plain text with
  our JSON envelopes.

## CLI surface

```
  --print, -p           Non-interactive JSON-lines output mode (Tier 2
                        #20). Emits one JSON object per line to stdout:
                        session_start, text, tool_use, tool_result,
                        session_end. Implies --quiet and --no-stream.
                        Suitable for piping into other tools.
```

## Honest trade-offs

- **One text event per turn, not per chunk.** Streaming JSON-of-chunks
  would be more responsive but harder to consume. Most automation
  callers want complete responses; one event per turn is the right
  level. If a future use-case needs token-by-token, we can add a
  `--print-stream` flag.
- **`tool_use` events fire BEFORE permission evaluation.** A denied
  call shows up as `tool_use` then a `tool_result` with
  `ok=false, result.error=...`. This is intentional: consumers see
  every model intention, not just successful ones, which matches
  Claude Code's `--output-format=stream-json` behavior.
- **No event for permission prompts.** Since `--print` is
  non-interactive, `ask` decisions get auto-declined (existing
  behavior from `is_repl=False` path). The `tool_result` shows the
  refusal reason. Future: add a `permission_decision` event.
- **`session_end` in the `finally:` block.** Fires even on
  exceptions, so consumers always see a terminating envelope.
  `wall_seconds` rounded to 3 decimal places (the metrics dict has
  full precision if a caller needs it via `--show-metrics`).
- **Broken-pipe tolerance.** If the consumer closes its stdin
  mid-stream, our `_emit_json` swallows the OSError so the agent
  loop can still terminate cleanly. Without this, `head -1 | open-code -p`
  would crash with `BrokenPipeError`.
- **Pyright noise.** ~20 new `Unknown` warnings (mostly through
  unannotated parameters in tests). Documented as cosmetic in the
  gap-log carry list; not blocking.
