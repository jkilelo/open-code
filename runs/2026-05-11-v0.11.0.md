## #4 Subagents / Task tool

A subagent is a named preset under `.open-code/agents/<name>.md`:

```
---
name: counter
description: A read-only subagent. Count and list things.
allowed-tools: list_dir, read_file
model: gemini-3.1-flash-lite-preview   # optional override
---
You are a focused counting subagent. Your only job: ...
```

A new tool `delegate(agent, task)` is exposed to the main loop:

```
delegate(agent="counter", task="count the .txt files")
  -> {ok: true,
      agent: "counter",
      subagent_session_id: "7d81cdc4-...",
      transcript_path: "...subagent.0.jsonl",
      summary: "There are 3 .txt files (a.txt, b.txt, c.txt).",
      exit_code: 0}
```

What happens under the hood when the main agent calls `delegate`:

1. Look up the agent definition under `.open-code/agents/`
2. Open a sibling JSONL transcript at
   `<parent>.subagent.<n>.jsonl` with a header that links back
   (`parent_session`, `agent_name`).
3. Recursively call `run_loop` with:
   - `system_instruction` = base + agent's body + task
   - `tool_allowlist` = agent's `allowed-tools` (if any)
   - `expose_delegate=False` -- **no recursion**
   - `verbose=False, stream=False` -- subagent output isn't shown to user
   - `max_iterations=10` (subagent budget, separate from parent)
4. Read the subagent's last model text as the `summary`.
5. Append a `delegate` event to the PARENT'S JSONL linking
   subagent_session_id + transcript_path + summary + exit_code.

The subagent gets a fresh `[]` history -- no leak from parent.

test 8 (`DELEGATE_TOOL_DECLARATION` shape) and implicit by the

## Honest trade-offs

- **`list_for_cwd` in SessionStore still globs `*.jsonl`** so subagent
  transcripts show up there too. They're filtered post-hoc by the
  user (you can tell from the `parent_session` field in the header).
  v0.12+ could add a filter. Documented in probe test 5 comment.
- **Subagent's transcript path is a sibling of parent's**, not under
  a subdir. This keeps the JSONL layout flat and makes
  parent-stem-prefix grouping work. If a session ever spawns 1000+
  subagents, this is a directory-listing issue; unlikely at our scale.
- **No streaming for subagents.** `verbose=False, stream=False` means
  the user doesn't see the subagent thinking. They see only the
  summary. This is by design (the whole point of subagents is to
  keep the main context clean) but power users may want
  `--show-subagent` flag later.
- **The `delegate` tool is special-cased in `run_loop`** rather
  than registered in `TOOL_FUNCTIONS`. This avoids passing
  parent state through tool args. Trade-off: any new tool needing
  parent context will need the same special casing.
