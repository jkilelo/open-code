## Three blockers closed

### 🔴 → ✅ #1 — apply_patch silent miswrite (corruption-class)

**Bug** (`patches.py:235`): anchor matching used substring `in`
comparison. `@@ def foo` matched both `def foo()` and `def
foo_helper()`. Combined with the rstrip-fallback's early `break`,
the patch silently applied to the wrong location with `ok: True`.

**Fix**:
- Anchor matching now uses `^<anchor>(?:\b|$)` — line starts with
  the anchor followed by a word-boundary character (or end of line).
  `@@ def foo` no longer matches `def foo_helper` because the
  character after `def foo` is `_` (a word character, no boundary).
- Ambiguous anchors (multiple matches) fail immediately with a
  clear error instead of silently picking the first match.
- The rstrip-fallback no longer breaks on the first match; it
  collects all matches and reports ambiguity.

**Probe** (`tests/probe_apply_patch_misanchor.py`):
```
CASE 1: ambiguous anchor 'def foo' (matches BOTH `def foo_helper` AND `def foo`):
  {'ok': False, 'error': "hunk 0: hunk pattern not found in file..."}
  after = (unchanged — file preserved)
  [PASS] Case 1: ambiguous anchor refused

CASE 2: rstrip-fallback when both lines rstrip-equal:
  {'ok': False, 'error': "hunk 0: hunk pattern is ambiguous — matches 2 locations..."}
  after = (unchanged)
  [PASS] Case 2: ambiguous rstrip-fallback refused
```

Original `probe_apply_patch.py` still 10/10 — the stricter matching
doesn't regress the happy path.

### 🔴 → ✅ #2 — MCP `_call` hang (no enforced timeout)

**Bug** (`mcp.py`): `CALL_TIMEOUT_SECS = 60` was defined at line 43
but never read. `_call`'s while-loop did blocking `readline()` with
no deadline. A buggy MCP server that ACKs `tools/call` but never
replies wedged the agent loop forever.

**Fix**:
- Per-server **reader thread** drains stdout into a `queue.Queue`.
- `_call` does `queue.get(timeout=...)` with a real wall-clock
  deadline derived from `CALL_TIMEOUT_SECS` (looked up at call time,
  so test code that mutates the module global gets the new value).
- On timeout, raises `TimeoutError` — `call_tool` catches it and
  returns `{ok: False, error: "TimeoutError: ..."}`.
- Reader thread shuts down via `srv.reader_stop.Event()` on
  `shutdown()`.

**Probe** (`tests/probe_mcp_hang.py`):
```
[mcp: connected to 'silent' with 1 tool(s)]
[PASS] call returned with timeout error: TimeoutError: MCP call 'tools/call' on server 'silent' timed out after 3.0s
OK -- MCP _call now respects CALL_TIMEOUT_SECS.
```

Probe sets `mcp.CALL_TIMEOUT_SECS = 3.0` then calls the tool against
a silent server. The call returns within 4s with a proper timeout
error instead of hanging.

Notable: the first fix attempt failed because of a Python default-
argument gotcha — `def _call(..., timeout=CALL_TIMEOUT_SECS)`
captured the value at function definition time, so mutating the
module global later had no effect. Fixed by using `timeout=None` as
default and reading `CALL_TIMEOUT_SECS` inside the function.

### 🟡 → ✅ #3 — PageRank personalization normalization

**Bug** (`repomap.py:213-215`): the personalization vector wasn't a
proper probability distribution. Personalized nodes got `1/|P|`,
non-personalized got `1/n`, total mass `|P|/|P| + (n-|P|)/n != 1`.
On a 5-node graph with `|P|=2`, scores summed to 1.6.

**Fix**: build the personalization vector `v[p]` as a real
probability distribution — all mass goes to nodes in P equally,
zero elsewhere. Teleport is `(1-d) * v[p]` and dangling-mass
redistribution also follows `v[p]` (personalized PageRank rule).

**Probe** (`tests/probe_pagerank_bug.py`):
```
=== no personalization ===
sum = 1.0000  (should be ~1.0)

=== personalization = {f0, f1} ===
sum = 1.0000  (PROPER PageRank should still be ~1.0)
[PASS] PageRank sum under personalization = 1.0000 (within 0.95-1.05)

=== ordering stability ===
top-3 without P: ['m9.py', 'm3.py', 'm5.py']
top-3 with P={m0}: ['m9.py', 'm0.py', 'm6.py']
```

Notice: now that personalization is correct, m0 actually appears
in top-3 when personalized on m0. Before the fix, personalization
had a weaker signal because mass was over-allocated.
