### #1 Hooks system

New module `hooks.py` (216 LOC). Five event types fire executable
scripts under `.open-code/hooks/<event>/`:

| Event | When | Block? | stdout JSON honored |
|-------|------|--------|---------------------|
| PreToolUse | Before each tool call | exit 2 = block call | `modifiedArgs`, `reason` |
| PostToolUse | After each tool call | never (observe-only) | (ignored) |
| Stop | After model's no-tool turn | exit 2 = force another iter | `reason` |
| SessionStart | First run_loop entry per session | (no effect) | `additionalContext` |
| UserPromptSubmit | Per REPL turn | exit 2 = skip turn | `transformedPrompt` |

Per Claude Code conventions:
- JSON document on stdin: `{event, session_id, cwd, ...event-specific}`
- Exit 0 = allow, exit 2 = block (reason from stderr or JSON.reason)
- Other exits logged and ignored (never crash the session)
- Env vars: `OPEN_CODE_PROJECT_DIR`, `OPEN_CODE_SESSION_ID`, `OPEN_CODE_CWD`
- 30-second timeout per hook
- Walks up from CWD to find `.open-code/hooks/`
- Scripts sorted by filename for deterministic ordering
- Cross-platform: `.py`/`.sh` work without +x; Windows accepts `.ps1`/`.bat`/`.cmd`/`.exe`

## Honest trade-offs

- **Hook scripts run with the same privileges as the user.** A
  malicious `.open-code/hooks/` could do anything. This matches
  Claude Code's posture — workspace trust applies. Document in
  the security section.
- **Hooks fire synchronously** and block the agent loop. A 30s
  timeout caps the worst case, but slow hooks make the agent feel
  slow. Async hooks could come later (Claude Code added `async:
  true` in May 2026 changelog — Tier 2 candidate).
- **No settings.json integration yet** (#2 in PROMPT-PACK). Hooks
  can't be disabled via configuration right now; the only way is
  to delete the script. #2 will add `permissions.hooks.disabled`.
- **PostToolUse can't actually block** in our implementation
  (matches Claude Code semantics). The probe documents that exit 2
  in PostToolUse "would block if honored — but we ignore."
- **Stop hook block message** is templated as
  `[Stop hook requested continuation: <reason>]`. Real-world
  reasons would benefit from richer formatting (markdown? JSON?);
  defer to v0.6 when usage patterns are clearer.
